Overview

Objective

The Code For Good mobile application serves to aid attendees of HackDuke's premiere hackathon by providing users with interactive schedules, check-in capabilities, directional maps, and a list of frequently asked questions.

My Role

Working with one other member of HackDuke Tech Team, we were responsible for creating the android version of the Code for Good application for HackDuke.

Development

A 3-week time crunch

Given less than 3 weeks to complete the app before the day of the hackathon, I began building our app's visual components in Android Studio's storyboard and writing code for the pages.

Staying flexible

With a short amount of time we had to prioritize features that would contribute the most to a minimum viable product over those that would complicate and lengthen development. We only included what was most essential to our attendees's user experience.

Features include:

  • Countdown to HackDuke date
  • Schedule for hackathon events on Saturday and Sunday
  • Interactive map that pinpoints an event's location when a user taps on an event
  • Help/FAQ menu for assistance and important contact numbers

Takeaways

Timely Manner

This was my first coding task coming to college and was taken aback by how short of a time span we had. However, I was more productive when working on the project knowing we had such a short time to complete it. I believe the app was done to the best of my abilities considering the timespan. If I were to do it again, I would have liked to include a live chat for attendees to ask questions they may have during the hackathon.

User-friendly

The simple design of the app was something I considered when developing the app. The app allows for easy navigation and understanding of the use. By only having one main page with a navigation bar directing to three other pages, it prevented any user from being confused where things were.

Git Version Control

Once again, this was my first coding project where I worked with more than just myself. This was a great introduction to using Git version control. I now know the essential git commands to do certain tasks such as checking out a repository, adding files, pushing to master, etc. Git allowed for me to keep track of all the changes in source code during the development.
